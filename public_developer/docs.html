NexusGuild Bot API

Build bots that interact with NexusGuild servers in real time. The API is designed to be familiar if you've worked with the Discord API — many concepts and endpoint paths are intentionally similar.

Overview

Bots communicate with NexusGuild in two ways:

  • Gateway — a persistent Socket.io connection that delivers real-time events (messages, interactions, member updates).
  • REST API — HTTP endpoints for reading and writing data (send messages, fetch members, manage reactions, etc.).

The base URL for all REST calls is:

https://www.nexusguild.gg/api/v1
Adding bots to servers: Only server owners and members with the Administrator permission can add bots. Manage Server alone is not sufficient. When a bot is authorized, a server role named after the bot is automatically created with the permissions the admin approved. The bot's effective permissions are determined by this role.

Authentication

All REST requests must include your bot token in the Authorization header:

Authorization: Bot YOUR_BOT_TOKEN

Your token is available on the Token tab of your bot in the developer portal. Treat it like a password — never expose it in client-side code or public repositories.

Note: NexusGuild bot tokens are plain hex strings, not the three-part JWT format used by Discord. The Bot prefix is still required.

Gateway

The gateway uses Socket.io (not a raw WebSocket). Connect to the /bot-gateway namespace and pass your token in the auth object.

Connecting

import { io } from 'socket.io-client';

const gateway = io('https://www.nexusguild.gg/bot-gateway', {
  auth: { token: 'YOUR_BOT_TOKEN' }
});

On successful authentication the gateway emits a READY event:

gateway.on('READY', ({ bot }) => {
  console.log('Connected as', bot.name, '/', bot.id);
});

Heartbeat

The gateway may send a HEARTBEAT ping. Respond immediately:

gateway.on('HEARTBEAT', () => gateway.emit('HEARTBEAT_ACK'));
Difference from Discord: Discord uses a raw WebSocket with numbered opcodes. NexusGuild uses Socket.io named events. If you're porting a Discord bot, replace ws / opcode logic with the Socket.io equivalent above.

Gateway Events

Your bot receives these events after connecting. The bot is automatically subscribed to all servers it has been added to.

EventWhen it firesKey fields
READY Immediately after authentication bot.id, bot.name
MESSAGE_CREATE A message is sent in any channel the bot can see id, channel_id, user_id, username, content, created_at
MESSAGE_UPDATE A message is edited id, channel_id, content
MESSAGE_DELETE A message is deleted messageId, channelId
INTERACTION_CREATE A user invokes a slash command registered to this bot id, token, channel_id, guild_id, user, data.name, data.options
MEMBER_JOIN A user joins a server the bot is in userId, username, serverId
MEMBER_LEAVE A user leaves a server the bot is in userId, serverId

Handling Slash Commands

gateway.on('INTERACTION_CREATE', async (interaction) => {
  const { id, token, data } = interaction;

  if (data.name === 'ping') {
    await fetch(
      `https://www.nexusguild.gg/api/interactions/${id}/${token}/callback`,
      {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ content: 'Pong!' }),
      }
    );
  }
});
Note: Interactions expire after 3 seconds. Respond to the callback URL within that window or the response will be ignored.

REST API

All endpoints accept and return JSON. Successful responses use 200 or 204. Errors return a JSON body with code and message fields, mirroring Discord's error format.

Users

GET /users/@me

Returns the bot's own user object.

{
  "id": "1234567890",
  "username": "MyBot",
  "avatar": null,
  "bot": true
}

Channels

GET /channels/{channel.id}

Returns a channel object. Requires VIEW_CHANNEL.

{
  "id": "...",
  "type": 0,
  "guild_id": "...",
  "name": "general",
  "topic": null,
  "position": 0,
  "parent_id": null
}

Channel type values mirror Discord: 0 = text, 2 = voice, 4 = category, 5 = announcement, 11 = thread, 15 = forum.

GET /channels/{channel.id}/messages

Returns an array of message objects, newest last. Requires VIEW_CHANNEL and READ_MESSAGE_HISTORY.

Query paramTypeDescription
limitintegerMax messages to return (1–100, default 50)
beforesnowflakeGet messages before this message ID
aftersnowflakeGet messages after this message ID
POST /channels/{channel.id}/messages

Send a message to a channel. Requires VIEW_CHANNEL and SEND_MESSAGES.

Body fieldTypeDescription
contentstringThe message text (required)
await fetch('https://www.nexusguild.gg/api/v1/channels/CHANNEL_ID/messages', {
  method: 'POST',
  headers: {
    'Authorization': 'Bot YOUR_TOKEN',
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({ content: 'Hello, world!' }),
});
PATCH /channels/{channel.id}/messages/{message.id}

Edit one of the bot's own messages. Requires VIEW_CHANNEL.

Body fieldTypeDescription
contentstringNew message text (required)
DELETE /channels/{channel.id}/messages/{message.id}

Delete one of the bot's own messages. Requires VIEW_CHANNEL. Returns 204 No Content on success.

PUT /channels/{channel.id}/messages/{message.id}/reactions/{emoji}/@me

Add a reaction to a message. Requires VIEW_CHANNEL and ADD_REACTIONS. The emoji must be URL-encoded (e.g. %F0%9F%91%8D for 👍). Returns 204.

DELETE /channels/{channel.id}/messages/{message.id}/reactions/{emoji}/@me

Remove the bot's reaction from a message. Returns 204.

Guilds

In NexusGuild, guilds are called servers — the IDs and concepts are identical.

GET /guilds/{guild.id}

Returns a guild object.

{
  "id": "...",
  "name": "My Server",
  "icon": null,
  "owner_id": "...",
  "approximate_member_count": 42
}
GET /guilds/{guild.id}/channels

Returns all channels in the guild as an array of channel objects.

GET /guilds/{guild.id}/members

Returns an array of member objects.

Query paramTypeDescription
limitintegerMax members to return (1–1000, default 100)
aftersnowflakePagination cursor (user ID)
{
  "user": { "id": "...", "username": "Alice", "avatar": null, "bot": false },
  "nick": null,
  "roles": ["role_id_1", "role_id_2"],
  "joined_at": "2025-01-01T00:00:00.000Z"
}
GET /guilds/{guild.id}/members/{user.id}

Returns a single member object.

Interactions

When a user runs a slash command registered to your bot, an INTERACTION_CREATE event is dispatched via the gateway. Respond by POSTing to the callback URL within 3 seconds.

POST /api/interactions/{interaction.id}/{interaction.token}/callback
Note: This endpoint does not use the /api/v1 prefix and does not require the Authorization header — the interaction token serves as authentication.
Body fieldTypeDescription
contentstringThe reply text to post in the channel

Registering Slash Commands

Commands are registered per-server from the Slash Commands tab in the developer portal. When a user types /commandname in a server where your bot is installed, they see an autocomplete suggestion. Selecting it dispatches an INTERACTION_CREATE to your bot.

Permissions

Bots are subject to the same permission system as regular users. Being a member of a server is not enough — the bot's roles must grant the required permissions for each action. Channel permission overrides (allow/deny per role or member) are also applied.

Required permissions per endpoint:

EndpointRequired permissions
GET /channels/:idVIEW_CHANNEL
GET /channels/:id/messagesVIEW_CHANNEL, READ_MESSAGE_HISTORY
POST /channels/:id/messagesVIEW_CHANNEL, SEND_MESSAGES
PATCH /channels/:id/messages/:idVIEW_CHANNEL (must be bot's own message)
DELETE /channels/:id/messages/:idVIEW_CHANNEL (must be bot's own message)
PUT .../reactions/:emoji/@meVIEW_CHANNEL, ADD_REACTIONS
DELETE .../reactions/:emoji/@meVIEW_CHANNEL
GET /guilds/:id and sub-resourcesBot must be a member of the guild

Bots with the Administrator permission bypass all channel-level checks.

Missing permission errors return 403 with code 50013 (Missing Permissions) or 50001 (Missing Access / VIEW_CHANNEL denied).

Quickstart

import { io } from 'socket.io-client';

const TOKEN = 'YOUR_BOT_TOKEN';
const API   = 'https://www.nexusguild.gg/api/v1';

// 1. Connect to the gateway
const gateway = io('https://www.nexusguild.gg/bot-gateway', {
  auth: { token: TOKEN }
});

gateway.on('READY', ({ bot }) => console.log('Ready:', bot.name));
gateway.on('HEARTBEAT', () => gateway.emit('HEARTBEAT_ACK'));

// 2. Listen for messages and respond
gateway.on('MESSAGE_CREATE', async (msg) => {
  if (msg.content === '!hello') {
    await fetch(`${API}/channels/${msg.channel_id}/messages`, {
      method: 'POST',
      headers: { 'Authorization': `Bot ${TOKEN}`, 'Content-Type': 'application/json' },
      body: JSON.stringify({ content: `Hello, ${msg.username}!` }),
    });
  }
});

// 3. Handle slash commands
gateway.on('INTERACTION_CREATE', async ({ id, token, data }) => {
  if (data.name === 'ping') {
    await fetch(`https://www.nexusguild.gg/api/interactions/${id}/${token}/callback`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ content: 'Pong!' }),
    });
  }
});
